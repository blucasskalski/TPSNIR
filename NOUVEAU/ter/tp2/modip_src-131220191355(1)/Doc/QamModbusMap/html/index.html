   Qam Modbus over TCP/IP Project (v2) Documentation

Le projet Qam Modbus over TCP/IP regroupe un ensemble de classes Qt/C++ destiné à la mise en oeuvre rapide d'applications Qt nécessitant l'implémentation de clients et/ou de serveurs sur un réseau Modbus. Le projet contient également des applications d'exemple de mise en oeuvre.

Modbus est un protocole permettant à deux entités d'échanger des données codées sur 1 bit ou sur 16 bits. Ces données primaires constituent la cartographie d'un équipement, elles sont reparties dans quatre tables :

  • table Coils : données 1 bit accessibles en lecture et écriture ;
  • table Discrete Inputs : données 1 bit en lecture seule ;
  • table Input Registers : données 16 bits en lecture seule ;
  • table Holding Registers : données 16 bits en lecture/écriture.

L'acès à une donnée se fait par son adresse dans sa table ; cette adresse est codée sur 16 bits.

Le répertoire ModipMap contient des classes permettant de modéliser d'un point de vue métier une cartographie Modbus. La vue métier permet de manipuler les données d'une cartographie non plus directement par leur adresse mais par un nom de rôle en rapport avec celui qu'elles occupent au sein de l'équipement.

Dans la vue métier, une donnée primaire 16 bits peut aussi se voir associer des données secondaires correspondant à un sous-ensemble de bits de la donnée primaire. Chaque donnée secondaire est aussi manipulable par son nom de rôle ; l'application métier n'a donc plus nécessité d'appliquer les techniques de bas niveau de masquage et de décalage... Plusieurs données primaires 16 bits peuvent aussi être associées pour former une donnée sur 32 bits, 64 bits ou plus. On parle alors de données composées.

La vue métier permet enfin de choisir un format pour chacune des données primaires, secondaires et composées ; elles deviennent alors manipulables en tant que grandeurs physiques (valeurs entières, valeurs réelles, chaines de caractères, ...).

Le répertoire ModipTcp propose un ensemble de classes relatives aux couches réseaux TCP/IP qui permettent de construire des serveurs multi-thread et des clients. Ces deux entités peuvent partager la même description de cartographie Modbus ; deux entités connectées, un client et un serveur, peuvent ainsi partager la même vue métier.
