ModipAsker

Application Qt client Modbus TCP/IP bas niveau

ModipAsker est un outil de test permettant d'établir une communication avec un équipement de type serveur Modbus TCP/IP. L'application offre à l'utilisateur la possibilité de fabriquer des trames Modbus au plus bas niveau, y compris des trames volontairement erronées à titre d'évaluation du traitement des erreurs par l'équipement distant.
Pour de plus amples renseignements sur la constitution des trames Modbus, voir l'extrait de documentation officielle Modbus Help Page.

Manuel utilisateur

L'interface utilisateur de ModipAsker est réduite à une unique boite de dialogue constituée de trois groupes de paramétrage et d'une console de suivi des échanges réseau.


Le bouton Clear permet d'effacer le contenu de la console de suivi d'exploitation.
Un clic sur le logo en haut à gauche provoque l'affichage d'une fenêtre "à propos..." montrant la version de l'application.

Internet Protocol

Ce groupe permet de fixer l'adresse IP du serveur à contacter et le port sur lequel est installé le service Modbus (normalement 502). Bien que l'application propose le choix entre les couches de transport TCP et UDP, les équipements industriels utilisent en général le mode connecté ; TCP est donc sélectionné par défaut.
Le bouton Connect permet de lancer la demande de connexion, le résultat est visible sur la console. Une fois la connexion établie, le bouton se transforme en Close et permet ainsi de mettre fin à la communication.

Modbus Application Protocol (MBAP)

Ce groupe sert à construire le header Modbus. L'utilisateur peut laisser les paramétrages par défaut (auto-incrémentation du numéro de transaction, identificateur de protocole, calcul automatique de la longueur de trame et identificateur d'esclave) ; mais il peut aussi volontairement modifier les paramètres à des fins de test.

Protocol Data Unit (PDU)

Ce groupe permet de choisir le type de requête (lecture/écriture de bits ou de mots), l'adresse de base des données à traiter, le nombre de données, et le cas échéant, les valeurs à transmettre. Les champs adresse, nombre et valeur peuvent être spécifiés en décimal ou en hexadécimal. Le champ Data utilisé par les requêtes d'écritures multiples est obligatoirement renseigné sous forme d'une suite d'octets hexadécimaux (avec ou sans séparateurs) ; logiquement, le nombre de symboles hexadécimaux de ce champ doit être égal à 4 fois la valeur indiquée par le champ Number.
Le bouton Send provoque l'émission effective de la requête à destination du serveur.
Losque l'application détecte une incohérence dans la trame à envoyer, elle le signale sous forme d'avertissement (affichage d'un message de couleur rouge sur la console), mais la trame est néanmoins envoyée au serveur.
Le bouton Send est utilisable hors connexion, ceci permet de visualiser les trames construites sans provoquer d'incidence dur le serveur.
Les trames sont montrées sur la console sous forme d'une succession d'octets exprimés en hexadécimal ; en vert pour l'émission et en bleu pour la réception.
