ModipManager

Application Qt client Modbus TCP/IP (vue métier)

ModipManager est une application permettant de matérialiser un maître Modbus susceptible de dialoguer avec plusieurs esclaves. L'application peut servir d'outil de test d'un processus automatisé ou de modèle de développement pour une application dédiée de contrôle-commande.
ModipManager permet de référencer plusieurs serveurs par l'intermédiaire, pour chacun d'eux, de leur fichier de configuration au format CSV (voir documentation de ModipButler).

Manuel utilisateur

L'interface utilisateur de ModipManager est réduite à une unique boite de dialogue occupée essentiellement par les vues arborescentes des cartographies des serveurs Modbus référencés.


Après démarrage de l'application, l'utilisateur doit charger au moyen du bouton New... les fichiers de configuration des serveurs Modbus requis. Chaque référencement de serveur provoque la création d'un nouvel onglet dans la zone des cartographies ; le titre de l'onglet correspond au texte du champ DESC du fichier de configuration. Le bouton Connect permet ensuite d'établir la connexion avec le serveur suivant les informations HOST et PORT du fichier de configuration (ces informations sont montrées dans les champs situés sous le bouton New..., elles peuvent être modifiées manuellement avant la connexion effective).
Tous les messages de suivi d'exploitation sont reportés dans la console située en haut de la fenêtre d'application.
La commande Reset all force localement à 0 les valeurs des tables, cette action est sans incidence sur le serveur distant. La commande Update all, disponible lorsque la connexion est active, provoque un cycle de lecture sur le serveur de toutes les entrées des tables.
L'utilisateur a la possibilité de lire individuellement chacune des données primaires, secondaires ou composées ; il suffit pour cela de sélectionner la ligne concernée dans la vue et de cliquer sur Update selection. Cette action entraine l'émission d'une requête de lecture à destination du serveur.
En tant que client, l'utilisateur peut modifier les valeurs du serveur accessibles en écriture (tables Coils et Holding Registers). L'édition se fait simplement par double-clic sur le champ valeur de la donnée ; la validation par Entrée entraine l'émission d'une requête d'écriture à destination du serveur.
Une fois la connexion établie, le bouton Connect se transforme en Close. Cette action permet de terminer la connexion.
Un clic sur le logo en haut à gauche provoque l'affichage d'une fenêtre "à propos..." montrant la version de l'application.
